# Phase P19: Public projects & direct doc links

_Intent: see [intent.md](intent.md)._

## Objective

Add per-project public/private visibility: public projects' docs and graph are readable by anonymous visitors and other users (the graph exposes only public-project nodes to outsiders), and every save returns a working direct URL to the doc — replacing the legacy mkdocs url field — shareable when the project is public.

## Context

Two paired capabilities (intent points 5 & 6): per-project public/private visibility (default private), and a save-time direct doc URL that replaces the broken legacy mkdocs `url`. Today there is **no visibility concept anywhere** and **no anonymous read path at all** on prod; the graph is session-only and tenant-wide. See `intent.md` for the confirmed intent.

## Decomposition

Five implementation middle slices, ordered by dependency. Risk selects the executor tier and is set deliberately (no `low` slices — none of this work is fully mechanical).

- **P19.S1 — Backend visibility core** (`medium`; order 1). Alembic `0004` adding `projects.visibility` (`Text NOT NULL DEFAULT 'private'`, `down_revision = "0003_org_level_credentials"`), plumbed through `ProjectModel` (`models.py:87`), `ProjectRecord` (`accounts/types.py:62`), `serialize_project` (`app_api.py:66`), and `AccountsRepository`/`CreateProject`. New `PATCH /app/projects/{project_id}` toggle (session-only, 404-cross-tenant per `_load_scoped_project`). `get_or_create_project` (`accounts/service.py:246`) defaults new rows to `private`. Terse tests. **Rationale:** a bounded, well-understood migration + CRUD + plumbing slice with no auth-boundary novelty — the data foundation every later slice reads. `medium` (not `low`) because it spans a migration, a new endpoint, and multi-layer plumbing that needs judgment on the toggle contract; not `high` because it introduces no new trust boundary.
- **P19.S2 — Backend public read surface** (`high`; order 2; depends P19.S1). The auth-boundary slice: a new **optional-identity** read dependency (authenticated tenant context when a session/bearer resolves, else anonymous), anonymous-capable `GET /documents/{id}` + raw (`/app/documents/{id}/raw`), and an **org-scoped public graph** endpoint filtered to the tenant's public projects. Implements the **Postgres→SQLite visibility bridge** (resolve the tenant's public-project-**name** set from Postgres per read, pass as an allowlist predicate into the SQLite read path — `db._filtered:256`, `get_document:224`, graph build). 404-never-403 for private/nonexistent. Terse tests incl. the negative: private docs/nodes must not leak anonymously. **Rationale:** highest risk — new trust boundary + cross-plane bridge + security-critical non-leak guarantees. `high` → `slice-executor-high`.
- **P19.S3 — Web public surfaces** (`high`; order 3; depends P19.S2). Visibility toggle on project detail (`projects/[projectId]/page.tsx:190`, reusing existing form/clipboard patterns) + dashboard row badge; public doc page and public graph page **outside the `(app)` gate** (new route group / self-guarding pages), composed 1:1 from existing document-detail rendering and `<GraphCanvas>`; a share/copy-link affordance (reuse mint-key clipboard pattern); `robots.ts`/`sitemap.ts` amendments to expose public surfaces. **Rationale:** establishes the web public/private boundary — a public route group outside the auth gate and an **anonymous BFF raw-HTML relay** that must preserve the P16 `sandbox`/CSP headers unchanged. A mistake here leaks private content or breaks sandboxing, so `high` despite being composition-only (no design authoring).
- **P19.S4 — Save-URL fix + CLI + skill + parity** (`medium`; order 4; depends P19.S1). Mode-aware `url` at the single build site (`main.py:592-595`): tenant mode → `{app origin}/documents/{id}` (origin from `KB_PUBLIC_BASE_URL`, already the app origin on prod); legacy mode → keep the mkdocs URL. CLI un-hides the URL (`cli/.../knowledge.py:402-410`, `guide.py:137`); both `SKILL.md` copies updated in byte-parity; `server/` additions mirrored into `plugin/templates/kb/` + `manifest.json`. Terse tests. **Rationale:** orthogonal to visibility but gated behind the parity CI (`plugin_parity.py`, `skills_parity.py`) and a mode-aware branch that needs judgment — `medium`, not `low`, because of the parity-sync discipline and the legacy-vs-tenant branch.
- **P19.S5 — Prod cutover** (`high`; order 5; depends S1–S4). Operator-gated deploy: reconcile box clone → `docker compose -f compose.prod.yml run --rm api alembic upgrade head` (runs `0004`) → `deploy/deploy.sh` → live public-link E2E smoke. Uses `pending` checkpoints (operator runs the ops actions). **Rationale:** ops orchestration + operator coordination + irreversible prod migration → `high`.

## Findings & Notes

**Fact corrections / clarifications (verified this slice; supersede loose wording in `intent.md`):**
- **`KB_REQUIRE_READ_AUTH` is a no-op in tenant mode.** It is consulted only in the legacy branch of `resolve_api_read` (`api_auth.py:227-234`); the tenant branch (`:236-244`) always requires a resolvable bearer regardless. `intent.md`'s phrasing ("no anonymous read path… `KB_REQUIRE_READ_AUTH=true`") is right about the outcome (no anonymous path) but the flag is not the cause in tenant mode — the tenant branch simply has no anonymous route. P19.S2 must **add** an anonymous route, not flip a flag.
- **No visibility column exists anywhere.** `projects` (`models.py:87-108`) is `id, tenant_id, name, created_at`, `UNIQUE(tenant_id,name)`. No PATCH route exists on projects (routes confirmed: GET/POST `/app/projects`, GET `/app/projects/{id}`, credential subroutes — no PATCH). `serialize_project` (`app_api.py:66-74`) and `ProjectRecord` (`types.py:62-68`) both need a new `visibility` field.
- **Save `url` is built at one site**, `main.py:592-595` = `KB_PUBLIC_BASE_URL/{project}/{date}-{slug}/` — legacy mkdocs, retired on prod at P14, wrong for all non-#1 tenants; CLI already hides it deliberately.
- **Cross-plane split:** project identity/visibility lives in **Postgres**; the SQLite `documents` table (`db.py:22-40`) carries `tenant_id` + project **name string** only (no FK, no visibility). Every doc/graph visibility filter must bridge name↔visibility per tenant. Graph node `url` is already `/documents/{id}` (`graph_api.py:83`) — the same shape the save-URL fix targets.
- **404-never-403** is the established cross-tenant convention (`app_api.py`, `documents_api.py`, `db.py:224`); public sharing extends it (private/nonexistent → 404).

**Design-stance decisions (rationale):**
1. **Visibility bridge → per-read resolution from Postgres** (not denormalize onto SQLite rows). Resolve the tenant's set of public project **names** from the small Postgres `projects` table per read, and pass it as an allowlist predicate into the SQLite read path. Rationale: the toggle takes effect **instantly** with no SQLite schema change and no reindex; visibility already lives in Postgres; the read set is tiny. Feasible at current scale — graph is bounded by `MAX_DOC_NODES` (`graph_api.py:59`, ≤2000 nodes) and the projects table is small, so a per-request lookup is negligible. (Revisit only if a public-read hot path emerges at much larger scale.)
2. **Public URL namespace → reuse existing shapes.** Docs: public read at the existing **`/documents/{id}`** shape (server gains an anonymous-capable read; web adds a public/optional-identity route) for share-link continuity — same URL authenticated or anonymous. Graph: an org-scoped public graph needs an **org identifier in the path**; **tenant UUID is acceptable for MVP**, an org **slug** is a deferrable nicety (note it as a later candidate). Private/nonexistent → **404, never 403**.
3. **Auth shape → optional-identity dependency** (not separate public endpoints) on the server: one read path that yields an authenticated tenant context (full tenant reads) or anonymous (public-only, scoped by the tenant addressed via the doc id / graph org param). On the web, public pages live **outside the `(app)` gate** (new route group or self-guarding pages); the raw-HTML relay must work **anonymously** while keeping the P16 `sandbox="allow-scripts"` + CSP headers unchanged.
4. **Save `url` is mode-aware.** Tenant mode → `{app origin}/documents/{id}` with the origin sourced from `KB_PUBLIC_BASE_URL` (already the app origin on prod). Legacy mode (the dormant single-tenant template stack, where mkdocs at `KB_PUBLIC_BASE_URL` still exists) → **keep** the mkdocs URL. CLI un-hides the URL; both `SKILL.md` copies and the CLI guide are updated; `api.md`/`backend.md` get doc-impact notes.

**Deferred / out-of-scope candidates (record, do not build):** org **slug** vanity URLs for the public graph (UUID suffices for MVP); any genuinely new visual chrome — branded share pages, an anonymous-visitor sign-up CTA — is a future design round, out of scope here.

**Cross-slice notes (P19.S2, done — the anonymous read surface / first trust boundary):**
- **Org selector = `?org=<tenant_uuid>` query param** on the existing `GET /app/graph` (settled the phase's open question; least routing churn, the bare logged-in call stays byte-compatible, and the graph payload carries no tenant id). The public graph view is `GET /app/graph?org={tenant_uuid}` (anonymous or cross-org); the bare `/app/graph` (no `org`) is unchanged and still 401s unauthenticated. Slug vanity URLs remain the deferred nicety.
- **S3-facing read contract (compose against these — shapes unchanged, now anonymous-capable):** public doc read = `GET /app/documents/{id}` (JSON, drops `tenant_id`) and `GET /app/documents/{id}/raw` (raw HTML + the four P16 sandbox headers, byte-identical); public graph = `GET /app/graph?org={tenant_uuid}` (only public-project nodes/edges/tag-hubs, `truncated` as usual, no org echo). A member's own org UUID for the web to build `?org=` is the tenant id from `/auth/me`. Everything private/nonexistent is an indistinguishable **404** (404-never-403); no `403` anywhere on this surface.
- **Optional-identity dependency** `optional_user` (`server/accounts/auth.py`): resolves a bearer exactly as `require_user` (incl. best-effort `last_used_at` stamp) but **returns `None` instead of raising** on every miss — never raises. Deliberate contract shift: an unresolvable token on a private doc now yields the anonymous **404**, not a 401 (leaks less; the BFF pre-verifies sessions). Only `GET /app/documents/{id}` + `/raw` + `/app/graph` swapped to it; `/app/documents` list and `/app/search` stay session-only (`require_user`) — so **two existing assertions were updated** to the new contract: `test_documents_require_auth` (`/app/documents/1` unauth 401→**404**; list+search still 401) and `test_html_documents::test_raw_route_serves_html_and_404s_for_md` (unauth raw 401→**404**, non-public doc). Not absorbed: **D15** (`test_documents_list_detail_and_project_bridge`) still fails on the P16 `format` list-projection drift — untouched.
- **Visibility bridge (Postgres→SQLite), per-read:** doc read — resolve `owner = row["tenant_id"]` (empty `''` legacy sentinel / unparseable UUID ⇒ never public ⇒ 404), then `get_project_by_name(owner_uuid, row["project"]).visibility == "public"`. Graph — `public_names = [p.name for p in list_projects_for_tenant(org) if p.visibility=="public"]`; empty (also covers a nonexistent org) ⇒ 404 (no existence leak). New allowlist predicate `db._filtered(..., projects: Sequence[str] | None)` → `project IN (?,…)`; default `None` leaves every existing caller byte-identical; an **empty** list fails closed (`AND 0`, matches nothing). **Legacy-mode guard** on both paths: `config.database_url() is None` ⇒ 404 before touching the (absent) accounts service, so the single-tenant template stack never 500s and never exposes a public path.
- **Test-harness gotcha (full gated suite):** the in-process auth rate-limiter (`KB_AUTH_RATE_LIMIT`, default **20** signups/IP/window) is shared across every Postgres-gated suite in one `pytest` process; `test_public_read.py` adds 6 signups, tipping the cumulative count over 20, so a **whole-suite** run 429s the last-alphabetical gated files (org_credentials, public_read). Run the full gated suite with `KB_AUTH_RATE_LIMIT=0` (or a high value) — the plan's 3-suite target command stays under the limit and passes 14/14. Not a code defect; CI runs only the parity checks (no pytest).

**Deferred / out-of-scope candidate (P19.S2):** **no anonymous-read rate limiting** — the new public doc/raw/graph surface has no per-IP throttle (only `/auth/*` is throttled). A future candidate: an edge or in-process limiter on the anonymous read paths before they see real traffic.

**Cross-slice notes (P19.S3, done — the web public/private boundary):**
- **Public web URL contract (for S4's save-URL fix + S5's E2E smoke):** a public project's doc is readable at the UNCHANGED **`/documents/{id}`** (now an optional-identity page in a new `(public)` route group, outside the `(app)` gate — same URL for member or anonymous); the org's public graph is at **`/graph/{org_uuid}`** (`org` = the tenant UUID from `/auth/me`; a member's own org id renders the full map, an anonymous/cross-org visitor the public subset). So S4's tenant-mode `url = {app origin}/documents/{id}` lands directly on this page. **Anonymous miss handling differs by surface:** an anonymous doc miss (private/nonexistent → server 404) **redirects to `/login`** (uniform, no returnTo); a public-graph miss (no public projects / nonexistent org → 404) renders a branded not-found (never a login bounce). A **malformed** id/org short-circuits to `notFound()` before any session read.
- **Optional-identity is now a web guard too:** `optionalIdentity()` (`web/src/lib/auth-guards.ts`) — the anonymous-capable sibling of `requireIdentity`: returns the member context for a live session, `null` for no-cookie OR a dead cookie (knowledge 401), rethrows real outages. It NEVER redirects. The raw-HTML relay (`/api/documents/{id}/raw`) is likewise now optional-identity: it forwards the bearer when a cookie is present, else relays **tokenless** (`openSession(...) ?? undefined`) — the anonymous browser sends no cookie, so it can never fetch-with-token on behalf of an anonymous visitor. The four P16 sandbox headers are byte-identical (relay test asserts the tokenless call carries no `Authorization` and the 200/404 mapping holds member-or-anonymous). BFF wrappers `getDocument`/`getDocumentRaw` now take `token: string | undefined`; `getGraph(token, { org? })` appends `?org=` (bare call unchanged); new `setProjectVisibility(token, id, visibility)` → PATCH; `getProject` already existed (reused).
- **Dashboard badge LANDED** (bounded discretion resolved): the `/app/dashboard` rollup row is built with a Postgres `ProjectRecord` in hand (`list_projects_for_tenant`, which carries `visibility`), so `dashboard_api.py` gained a `"visibility"` key (canonical + byte-mirrored to `plugin/templates/kb/server/dashboard_api.py`; `plugin_parity.py` PASS), `KbDashboardProject.visibility` was added, and the dashboard projects table gained a Public/Private badge column.
