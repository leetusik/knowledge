# P12.S2 — Plan (Auth + BFF proxy + authenticated app shell)

Orchestrator's native plan for **`P12.S2`** (`high` risk), executed by `slice-executor-high`. The executor implements against the sources below, writes `result.md`, appends a `phase.md` note, returns a verdict. It **never commits** and **never transitions status**.

## Context

S2 is the **security-sensitive core** of the web app and its whole server-side data layer. It builds: a typed server-side knowledge API client, the **BFF auth flow** (signup/login/logout) with a **sealed httpOnly cookie**, page guards, and the **authenticated app shell** that S3–S6 render inside. The browser talks only to the Next origin; Next calls `knowledge-api` server-to-server with `Authorization: Bearer`; the backend token is sealed into an AES-256-GCM httpOnly cookie — **no backend CORS change, no web-side DB**.

Two demands, same split as S1:
- **Functional/security layer → port from vocky (`~/projects/personal/vocky/web/src/`) faithfully** (it is proven + fully unit-tested). Knowledge's backend already speaks vocky's **exact** `/auth/*` contract (verified in phase.md anchors): `POST /auth/signup`→`201 {token,user,tenant}` (singular), `POST /auth/login`→`200 {token,user,tenants[]}` (plural), `GET /auth/me`→`{user,tenants[]}`, `POST /auth/logout`→`204`, generic 401 (no enumeration), opaque 30-day `token_urlsafe(32)` bearer. So the port precondition holds — no client contract changes.
- **Design skin → from hi2vi_web's authenticated area (`~/projects/personal/hi2vi_web/src/app/operator/`)**, per the operator's "whole pages like hi2vi_web" directive. Vocky's shell/auth pages carry the same *structure* but its neutralized look; re-skin to hi2vi_web's real design.

## Part A — Functional/security layer (port from vocky, adapt identity)

Build **bottom-up** in this order (each satisfies the next's imports):

1. **`lib/env.ts`** (`server-only`, lazy per-consumer reads so `next build` needs no env): `KB_API_BASE_URL` + `SESSION_SECRET` (neither `NEXT_PUBLIC_`; unset-or-blank = missing; errors name keys never values).
2. **`lib/session.ts`** (`server-only`, `node:crypto`) — the sealed cookie, **port verbatim** except the cookie name. Constants: cookie **`knowledge_session`**; TTL 30 d (`Max-Age=2592000`); envelope `v1.<iv>.<ct>.<tag>` base64url; **AES-256-GCM**, key `sha256(SESSION_SECRET)`, IV `randomBytes(12)`, 16-byte tag; payload `{token, exp}`. Exports `sealSession`, `openSession` (returns `null` for **any** failure incl. tamper/expiry/unset secret — never throws), `getSession`, `requireSession` (→ `redirect("/login")`), `readSessionCookie(req)`, `assertSameOrigin(req)`. Cookie attrs `{httpOnly:true, sameSite:"strict", secure: NODE_ENV==="production", path:"/", maxAge:2592000}` — **secure prod-only** (dev http/127.0.0.1 would drop a Secure cookie).
3. **`lib/knowledge/{types,client,auth}.ts`** (renamed from vocky's `lib/vocky/*`; `Kb*` type prefix). `client.ts`: `ApiError{status,detail}`, `getJson`/`getRaw`/`sendJson`, absolute URL from `KB_API_BASE_URL` (trailing slash stripped), `Authorization: Bearer` injection, **every call `cache:"no-store"`**, 204→undefined. `auth.ts`: `activeTenant(raw)=raw.tenant ?? raw.tenants?.[0] ?? null` + `normalizeAuthResponse` (collapses the singular/plural asymmetry to `tenants[0]`), `signup`/`login`/`logout`(fire-and-forget)/`me`. `types.ts`: **S2 subset only** — `KbUser`, `KbTenant`, `KbSession`, `KbIdentity` (ids are strings, timestamps ISO). (Do **not** port `app.ts` or the S3+ types — S3 adds them.)
4. **`lib/client-ip.ts`** + **`lib/rate-limit.ts`** — port verbatim (per-process in-memory sliding window; the cross-window prune-by-own-`windowMs` fix is load-bearing).
5. **`lib/bff.ts`** (`server-only`) — the audited pipeline, ordered stages **415 → 403 (same-origin) → 429 (rate-limit *before* parsing) → 400 (bad JSON) → 422 (zod: email min1 max320, password min1 max1024) → 502 (backend `ApiError` off the pass-through set) / 500 (transport/seal) → 200**. Body is always `{ok}` only; **backend `detail` never echoed** (preserves enumeration-safety); rate key `` `${bucket}:${ip}` ``; success `sealSession` + `Set-Cookie`.
6. **`lib/auth-guards.ts`** — `requireIdentity = cache(async …)` (`requireSession` → live `GET /auth/me`; **catch: only `ApiError` 401 → `redirect("/login")`, everything else rethrown** so an outage ≠ logout); `redirectIfAuthenticated` (re-verifies against `/auth/me` to break the **revoked-cookie ping-pong**).
7. **`app/api/auth/{login,signup,logout}/route.ts`** — all `runtime="nodejs"` + `dynamic="force-dynamic"`. login `{limit:5, windowMs:15*60000, bucket:"auth-login", call:login, passThroughStatuses:[401,400]}`; signup `{10, …, "auth-signup", signup, [409,400]}`; logout = own handler (`assertSameOrigin` → best-effort `logout(token)` → **unconditional** cookie clear; never fails open).
8. **`app/(auth)/{login,signup}/*` + `auth-card.tsx` + `credentials-form.tsx`** — `(auth)/layout.tsx` **unguarded** (login must stay reachable) + `robots:noindex`; pages `await redirectIfAuthenticated()` then render the card + form. `credentials-form.tsx` (`"use client"`): plain `fetch` to the BFF route (**not** `useActionState`), reads **status only** (never body), on `res.ok` → `router.replace("/dashboard")+router.refresh()`, on fail → `errorFor(status)` + clear password. `errorFor`: login `401→invalidCredentials` (one message for unknown-email AND bad-password — no enumeration), `400|422→invalidInput`, `429→rateLimited`; signup `409→emailTaken`, …
9. **`components/app-shell/*`** — `app-shell.tsx` (**server component**, props `{identity, children}`); `rail-nav.tsx` (`"use client"` only for `usePathname` active state + `soon` muted "Soon" pills, never a 404 link); `logout-button.tsx` (`"use client"`, POST `/api/auth/logout` then `router.replace("/login")+refresh()` in `finally`); barrel `index.ts`.
10. **`app/(app)/layout.tsx`** (`const {identity} = await requireIdentity()` → `<AppShell>`), a **minimal** `app/(app)/dashboard/page.tsx` placeholder (heading + signed-in email/tenant — just proves the gate renders; S3 swaps real data), and **`app/page.tsx`** → root redirect (`getSession()` ? `/dashboard` : `/login`) **replacing S1's preview**.
11. **`content/auth.ts`** (`AUTH_ERRORS`, `LOGIN_PAGE`, `SIGNUP_PAGE`) + **`content/app.ts`** (`APP_NAV`, `APP_SHELL`); extend `content/index.ts` barrel. `APP_NAV` = **Dashboard** (live) + **Documents** & **Graph** as `soon:true` (they ship in S5/S6 — the shell shows the full intended nav with "Soon" pills). No top-level Projects (reached from the dashboard).
12. **`tests/*` + `tests/stubs/server-only.ts` + `vitest.config.ts`** — port the vitest suites (`session`, `session-guards`, `auth-routes`, `knowledge-auth`), aliasing `server-only`→the inert stub and `@`→`./src`. Keep them terse (the crypto + BFF-pipeline + tenant-normalization cases are the high-value ones).

**Load-bearing gotchas (do not "optimize" away):** `requireIdentity` rethrows non-401; `(auth)` pages re-verify to break the ping-pong; guards can't clear a stale cookie (login overwrites / logout clears); GCM tag = integrity (keep length+tag checks); rate-limit before parsing; distinct per-route bucket; `cache:"no-store"` everywhere; `server-only` imports guard the whole server layer (client import → build fail); backend `detail` never echoed; logout best-effort revoke + unconditional clear.
