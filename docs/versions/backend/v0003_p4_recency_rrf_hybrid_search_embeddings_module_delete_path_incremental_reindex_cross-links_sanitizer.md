# Backend

## Status

Implemented and validated (Track 2). The `server/` package provides the DB-backed document API. `docs/` is canonical; the SQLite DB is disposable. P4 added: recency-aware, paginated, hybrid (keyword + Gemini vector) search; a new `embeddings` module; a symmetric document-delete write path; tag/project aggregations; incremental single-path reindex + startup drift self-heal; a `related` cross-link convention; and a publish-safe `source_repo` sanitizer — all additive and backward-compatible.

## Stack

- Language/runtime: Python 3.12.
- Framework: FastAPI + uvicorn.
- Package manager: uv — **virtual project** (`[tool.uv] package=false`, no `[build-system]`): deps only, never built/installed (matches the container's `--no-emit-project`). `[tool.pytest.ini_options] pythonpath=["."]` lets `import server` resolve without installing.
- Server entrypoint: `server.main:app` (`uvicorn server.main:app`), **single worker** (load-bearing — see invariants).
- Deps: `fastapi`, `uvicorn`, `pyyaml`, `google-genai` (embeddings, reusing changple5's pin); dev: `pytest`, `httpx`.

## Module Layout (`server/`)

- **`config`** — env-at-call-time settings (no import-time caching, so tests retarget per call): `KB_ROOT` (default cwd), `docs_root` (=`KB_ROOT/docs`), `KB_DB_PATH` (default `KB_ROOT/data/kb.sqlite3`), `KB_PUBLIC_BASE_URL` (default `http://localhost:8765`, the viewer origin for response `url`s), `KB_API_TOKEN` (unset by default), `KB_GIT_COMMIT` (default true), plus (P4) `gemini_api_key` (`GOOGLE_API_KEY` preferred → `GEMINI_API_KEY` fallback; `None` = semantic search off), `embedding_model` (`GEMINI_EMBEDDING_MODEL`, default `gemini-embedding-2-preview`), `embeddings_enabled`, and `startup_reindex_enabled` (`KB_STARTUP_REINDEX`, default true).
- **`db`** — `connect(path=None)` (WAL, `sqlite3.Row` factory, `PRAGMA foreign_keys=ON`, idempotent DDL + an idempotent `PRAGMA table_info`→`ALTER TABLE` migration for the P4 `related` column, creates parent dirs); `upsert_document` (ON CONFLICT(rel_path); preserves `created_at`, refreshes `updated_at`; now threads `related`), `get_document`, `get_document_by_path`, `list_documents` (newest-first, tag via `json_each`), `count_documents`, `delete_document_by_path` (relies on the FTS `AFTER DELETE` trigger + the embedding FK cascade — no extra cleanup code). P4 aggregations: `list_tags(conn, project=None)` and `list_projects(conn)` (both `json_each`/`GROUP BY`, pre-ordered for display). P4 embedding cache: `document_embeddings` DDL + `upsert_embedding`, `get_embedding`, `get_embedding_hashes`, `get_all_embeddings`, `delete_orphan_embeddings`. Reads JSON-decode `tags` and `related` to lists.
- **`documents`** — conventions + write composition: `slugify`; validators `validate_project/tags/slug/date` (raise `ConventionError`; `FrontmatterError` is a subclass — both map to 422), plus P4 `validate_related` (list of relative, no-`..`, ≥2-part, `.md` rel_paths; dedup order-preserving; `ConventionError` otherwise); `rel_path`; **hand-rolled byte-exact** `serialize_frontmatter` (title via `json.dumps(ensure_ascii=False)`; bare date; tag list; optional two-space-indented `related:` block between `tags:` and `source:`, emitted only when non-empty so absent-`related` output stays byte-identical; `source:` map) with `parse_frontmatter`; `insert_recent_bullet` / `update_recent_index` and their P4 inverses `remove_recent_bullet` (pure — drops the line carrying `](<rel_path>)`) / `remove_from_recent_index` (I/O; idempotent, no write when nothing matched); `write_document_file` (now takes optional `related`; returns the body **exactly as reindex would store it** — no drift); P4 `sanitize_source_repo` (local paths → basename, URLs pass through, empty for null).
- **`embeddings`** (P4, new) — the **only** module that talks to Gemini (`google-genai`, model `gemini-embedding-2-preview`): `embed_texts(texts, *, kind, retries)` (one request per text — the SDK does not batch; bounded 429 backoff used only on the reindex path), `document_input`, `content_hash` (sha256 of model + `title\n\nbody` truncated to `MAX_INPUT_CHARS=20000`), `pack_vector`/`unpack_vector` (L2-normalized float32 BLOBs, no numpy), `cosine`, and `EmbeddingError`. Vectors are 3072-dim.
- **`search`** — `build_match_query(q)` double-quotes each whitespace token (doubling internal `"`) and (P4) prefix-expands CJK/Hangul/Kana tokens to `"tok"*` so those scripts are word-searchable at the query layer without a tokenizer change; `search(conn, q, *, project, tag, limit=10, offset=0, raw=False)` re-ranks the **full** match set in Python — weighted BM25 (8/4/1) fused with an exp-decay recency signal from `d.date` (`score = bm25 + RECENCY_WEIGHT·recency`; `HALF_LIFE_DAYS=90`, `RECENCY_WEIGHT=0.5`) — then, when embeddings are enabled and the query embeds, fuses a cosine vector ordering (over `get_all_embeddings`) via **RRF** (`RRF_K=60`) over the union of both lists. Returns `{results, total, mode}` (`mode` = `"hybrid"`/`"bm25"`; `total` = match/fusion count); `signals` = `{bm25?, recency, vector?}`; `offset`/`limit` slice the composed order. `raw=True` and the no-vector/degraded paths stay exact BM25 behavior; a raw syntax error re-raises as `SearchQueryError` (→ 400).
- **`gitops`** — `GitError(command, stderr)`; `add(paths, *, root)` = `git -C <root> add -- <paths…>` (only the touched paths — **never `-A`**); `commit(msg, *, root, co_authored_by=None)` → `rev-parse HEAD` (optional second `-m` trailer). Every failure (nothing-to-commit, missing identity, unsafe dir) raises `GitError`; **never pushes**.
- **`reindex`** — library fns + `python -m server.reindex [rel_path]` CLI; `reindex()` walks `docs/<subdir>/**/*.md`, upserts present rows, removes vanished ones (`RESERVED_DIRS = {"current","versions"}` excluded; malformed → `skipped[]`), then runs `_sync_embeddings(conn)` (content-hash-cached, per-doc incremental embed+upsert, orphan/model-mismatch cleanup — best-effort, never fails the reindex) and folds its report under `embeddings:`. P4 `reindex_path(rel_path, conn=None, docs_root=None)` validates the rel_path shape (ValueError on absolute/`..`/<2-part/reserved/non-`.md`) and indexes or deletes a single path, running the same embedding sync. `_index_file` reads `meta.get("related")` and sanitizes `source.repo` leniently. Never runs git.
- **`main`** — FastAPI `app`; per-request `get_conn()` dependency; `require_bearer` dependency; `_public_doc` response shaper (drops `tags_text`; `markdown` only on single-doc fetches; passes through `related`); a P4 `lifespan` that runs a full `reindex()` on boot when `startup_reindex_enabled()` (drift self-heal, printing `[kb-api] startup reindex: …`); module-level `WRITE_LOCK = threading.Lock()` wrapping the write critical section of `POST /api/documents` **and** the P4 shared `_delete_document` helper (reused by both DELETE routes: file remove → Recent-bullet removal → `db.delete_document_by_path` → scoped commit, same no-rollback contract). `create_document` validates/drops-self-ref `related`, sanitizes `source_repo`, and **best-effort embeds the new doc outside `WRITE_LOCK`** (any failure swallowed — never affects the 201). Adds the open `GET /api/tags`/`GET /api/projects` reads and the `ReindexIn`-bodied `POST /api/reindex`.

## Invariants

- **Single-writer**: the write lock is **in-process**, so the API must run **one uvicorn worker** — never scale workers. WAL still gives read concurrency. P4's embedding work introduces no background workers — embeds happen in-request (best-effort, outside the lock) or at reindex, so the single-worker invariant is untouched.
- **`docs/`-canonical, never-rollback**: a failed git commit leaves the file/DB written (`committed:false`); `docs/` stays canonical and reindex reconciles. Both the POST write path and the P4 DELETE path share this contract. Git stages only the touched paths and never pushes.
- **DB/file consistency**: a POST-written document and a later reindex produce identical stored `markdown` — no phantom drift. `related` and the sanitized `source_repo` round-trip identically through the write path and reindex.
- **Embeddings are a disposable cache, best-effort everywhere**: no Gemini key → skipped; an API/429 failure → per-doc skip, reported and retried next reindex — never fails a reindex, a write, or a search (which degrades to BM25-only).

## Error Handling

- `ConventionError`/`FrontmatterError` (incl. `validate_related` shape) → 422; an invalid `reindex_path` rel_path → 422; existing target → 409 (names the doc); a missing delete target → 404; FTS5 raw-syntax error → 400 (`SearchQueryError`); git failure → 201/200 with `committed:false` + `commit_error`.
