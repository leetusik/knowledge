# Backend

## Status

Implemented and validated (Track 2). The `server/` package provides the DB-backed document API. `docs/` is canonical; the SQLite DB is disposable. P4 added: recency-aware, paginated, hybrid (keyword + Gemini vector) search; a new `embeddings` module; a symmetric document-delete write path; tag/project aggregations; incremental single-path reindex + startup drift self-heal; a `related` cross-link convention; and a publish-safe `source_repo` sanitizer — all additive and backward-compatible. P7 (F1) added a small write-path side effect: the first document of a **new project** also auto-creates that project's `docs/<project>/index.md` landing page (never overwriting an existing one), so every project satisfies the per-project deploy-gate invariant — also additive and backward-compatible. **P10** adds a second plane in the same process: an **async Postgres accounts/tenancy control plane** (`server/persistence/` + `server/accounts/`) plus tenant-scoping of the content plane (`server/api_auth.py`, tenant-filtered queries, a per-tenant content root). It is **lazy/dormant without `DATABASE_URL`** — the content plane behaves byte-for-byte as before (65-test legacy regression green) — and turns on only in tenant mode. **P11** adds **per-tenant usage metering**: a new `server/usage/` package (the durable `usage_events` event log + a derive-on-read aggregate service + a best-effort metering hook), a `server/usage_api.py` read router, and a **stash + async HTTP middleware** seam that lets the sync content handlers meter into async Postgres. It is **best-effort and inert in legacy mode** (65-test legacy regression still green) — a metering failure never changes a request's outcome. **P16** adds a first-class **HTML explainer document format** to the content plane, **additively**: `documents.py` gains an HTML comment-frontmatter serializer/parser + a stdlib `html.parser` text extractor; `db.py` gains `format`/`raw_html` columns; `main.py`/`reindex.py`/`seed.py` widen every hardcoded `.md` point to `.md`/`.html` and apply a shared *body rule*; and a new session-guarded `GET /app/documents/{id}/raw` route serves the raw HTML with sandbox headers. Markdown docs stay **byte-identical** (default `format="md"`, `_normalize_body` unchanged) and every change is additive. See *HTML explainer document format (P16)* below. **P18** reshapes the accounts plane for the user → org → project model: a new `provision_signup` service primitive (default org + default project + owner membership, atomic) that signup and fresh-DB seed both build on; the `/api/*` resolver now reads the credential's own `tenant_id` (org-level keys carry `project_id NULL`); a race-safe `get_or_create_project` + `list_org_credentials` on the accounts service; and an **async** write-path dependency (`ensure_registry_project`) that get-or-creates the Postgres `projects` row in front of the sync `WRITE_LOCK` handler. All additive, and `server/**` edits are mirrored into `plugin/templates/kb/` (parity). See *Accounts v2: org-level credentials + get-or-create (P18)* below.

## Stack

- Language/runtime: Python 3.12.
- Framework: FastAPI + uvicorn.
- Package manager: uv — **virtual project** (`[tool.uv] package=false`, no `[build-system]`): deps only, never built/installed (matches the container's `--no-emit-project`). `[tool.pytest.ini_options] pythonpath=["."]` lets `import server` resolve without installing.
- Server entrypoint: `server.main:app` (`uvicorn server.main:app`), **single worker** (load-bearing — see invariants).
- Deps: `fastapi`, `uvicorn`, `pyyaml`, `google-genai` (embeddings, reusing changple5's pin); **(P10)** `sqlalchemy` (2.0), `psycopg[binary]` (async psycopg3 driver), `alembic`, `argon2-cffi`, `greenlet`; dev: `pytest`, `httpx`.
- **(P10) Async control plane in a sync app.** The accounts plane is `async` (async SQLAlchemy 2.0 + `postgresql+psycopg`) and lives **alongside** the existing sync content endpoints in the one FastAPI app. The async-vs-sync integration Open Question was resolved to **async SQLAlchemy + psycopg3** (not asyncpg): a single worker means no async-throughput requirement, so the choice was made for the cleaner integration. The engine is a **lazy singleton** created on first use and disposed in the app lifespan; `DATABASE_URL` unset → no engine is ever created (accounts dormant).

## Module Layout (`server/`)

- **`config`** — env-at-call-time settings (no import-time caching, so tests retarget per call): `KB_ROOT` (default cwd), `docs_root` (=`KB_ROOT/docs`), `KB_DB_PATH` (default `KB_ROOT/data/kb.sqlite3`), `KB_PUBLIC_BASE_URL` (default `http://localhost:8765`, the viewer origin for response `url`s), `KB_API_TOKEN` (unset by default), `KB_GIT_COMMIT` (default true), plus (P4) `gemini_api_key` (`GOOGLE_API_KEY` preferred → `GEMINI_API_KEY` fallback; `None` = semantic search off), `embedding_model` (`GEMINI_EMBEDDING_MODEL`, default `gemini-embedding-2-preview`), `embeddings_enabled`, and `startup_reindex_enabled` (`KB_STARTUP_REINDEX`, default true).
- **`db`** — `connect(path=None)` (WAL, `sqlite3.Row` factory, `PRAGMA foreign_keys=ON`, idempotent DDL + an idempotent `PRAGMA table_info`→`ALTER TABLE` migration for the P4 `related` column, creates parent dirs); `upsert_document` (ON CONFLICT(rel_path); preserves `created_at`, refreshes `updated_at`; now threads `related`), `get_document`, `get_document_by_path`, `list_documents` (newest-first, tag via `json_each`), `count_documents`, `delete_document_by_path` (relies on the FTS `AFTER DELETE` trigger + the embedding FK cascade — no extra cleanup code). P4 aggregations: `list_tags(conn, project=None)` and `list_projects(conn)` (both `json_each`/`GROUP BY`, pre-ordered for display). P4 embedding cache: `document_embeddings` DDL + `upsert_embedding`, `get_embedding`, `get_embedding_hashes`, `get_all_embeddings`, `delete_orphan_embeddings`. Reads JSON-decode `tags` and `related` to lists.
- **`documents`** — conventions + write composition: `slugify`; validators `validate_project/tags/slug/date` (raise `ConventionError`; `FrontmatterError` is a subclass — both map to 422), plus P4 `validate_related` (list of relative, no-`..`, ≥2-part, `.md` rel_paths; dedup order-preserving; `ConventionError` otherwise); `rel_path`; **hand-rolled byte-exact** `serialize_frontmatter` (title via `json.dumps(ensure_ascii=False)`; bare date; tag list; optional two-space-indented `related:` block between `tags:` and `source:`, emitted only when non-empty so absent-`related` output stays byte-identical; `source:` map) with `parse_frontmatter`; `insert_recent_bullet` / `update_recent_index` and their P4 inverses `remove_recent_bullet` (pure — drops the line carrying `](<rel_path>)`) / `remove_from_recent_index` (I/O; idempotent, no write when nothing matched); `write_document_file` (now takes optional `related`; returns the body **exactly as reindex would store it** — no drift); P4 `sanitize_source_repo` (local paths → basename, URLs pass through, empty for null). **P7 (F1)** `project_landing_content(project)` (the minimal landing body: H1 = project name + a one-line description, **no YAML frontmatter** — so it stays a *non-doc*: on `graph_hook`'s `_SKIP_NAMES`, excluded from `discover_projects` doc-counting and from `check_graph`'s `fs_count`, existing only to produce the per-project `site/<project>/index.html`) and `ensure_project_landing(docs_root, project) -> bool` (writes it iff `docs/<project>/index.md` is absent; returns `True` when it created one, never overwriting an existing hand-written or auto-created landing).
- **`embeddings`** (P4, new) — the **only** module that talks to Gemini (`google-genai`, model `gemini-embedding-2-preview`): `embed_texts(texts, *, kind, retries)` (one request per text — the SDK does not batch; bounded 429 backoff used only on the reindex path), `document_input`, `content_hash` (sha256 of model + `title\n\nbody` truncated to `MAX_INPUT_CHARS=20000`), `pack_vector`/`unpack_vector` (L2-normalized float32 BLOBs, no numpy), `cosine`, and `EmbeddingError`. Vectors are 3072-dim.
- **`search`** — `build_match_query(q)` double-quotes each whitespace token (doubling internal `"`) and (P4) prefix-expands CJK/Hangul/Kana tokens to `"tok"*` so those scripts are word-searchable at the query layer without a tokenizer change; `search(conn, q, *, project, tag, limit=10, offset=0, raw=False)` re-ranks the **full** match set in Python — weighted BM25 (8/4/1) fused with an exp-decay recency signal from `d.date` (`score = bm25 + RECENCY_WEIGHT·recency`; `HALF_LIFE_DAYS=90`, `RECENCY_WEIGHT=0.5`) — then, when embeddings are enabled and the query embeds, fuses a cosine vector ordering (over `get_all_embeddings`) via **RRF** (`RRF_K=60`) over the union of both lists. Returns `{results, total, mode}` (`mode` = `"hybrid"`/`"bm25"`; `total` = match/fusion count); `signals` = `{bm25?, recency, vector?}`; `offset`/`limit` slice the composed order. `raw=True` and the no-vector/degraded paths stay exact BM25 behavior; a raw syntax error re-raises as `SearchQueryError` (→ 400).
- **`gitops`** — `GitError(command, stderr)`; `add(paths, *, root)` = `git -C <root> add -- <paths…>` (only the touched paths — **never `-A`**); `commit(msg, *, root, co_authored_by=None)` → `rev-parse HEAD` (optional second `-m` trailer). Every failure (nothing-to-commit, missing identity, unsafe dir) raises `GitError`; **never pushes**.
- **`reindex`** — library fns + `python -m server.reindex [rel_path]` CLI; `reindex()` walks `docs/<subdir>/**/*.md`, upserts present rows, removes vanished ones (`RESERVED_DIRS = {"current","versions"}` excluded; malformed → `skipped[]`), then runs `_sync_embeddings(conn)` (content-hash-cached, per-doc incremental embed+upsert, orphan/model-mismatch cleanup — best-effort, never fails the reindex) and folds its report under `embeddings:`. P4 `reindex_path(rel_path, conn=None, docs_root=None)` validates the rel_path shape (ValueError on absolute/`..`/<2-part/reserved/non-`.md`) and indexes or deletes a single path, running the same embedding sync. `_index_file` reads `meta.get("related")` and sanitizes `source.repo` leniently. Never runs git.
- **`main`** — FastAPI `app`; per-request `get_conn()` dependency; `require_bearer` dependency; `_public_doc` response shaper (drops `tags_text`; `markdown` only on single-doc fetches; passes through `related`); a P4 `lifespan` that runs a full `reindex()` on boot when `startup_reindex_enabled()` (drift self-heal, printing `[kb-api] startup reindex: …`); module-level `WRITE_LOCK = threading.Lock()` wrapping the write critical section of `POST /api/documents` **and** the P4 shared `_delete_document` helper (reused by both DELETE routes: file remove → Recent-bullet removal → `db.delete_document_by_path` → scoped commit, same no-rollback contract). `create_document` validates/drops-self-ref `related`, sanitizes `source_repo`, and **best-effort embeds the new doc outside `WRITE_LOCK`** (any failure swallowed — never affects the 201). **P7 (F1):** inside `WRITE_LOCK`, right after `write_document_file`, it calls `ensure_project_landing(docs_root, project)`; when that returns `True` the scoped commit stages a **third** path (`docs/<project>/index.md`) and the 201 body carries `landing_created:true` (else the usual 2 paths, `landing_created:false`). The DELETE path is deliberately unchanged: deleting a project's last real doc leaves the auto-landing behind, but `discover_projects` no longer lists a project whose only file is `index.md`, so the gate stays satisfied — no delete-side cleanup is warranted. Adds the open `GET /api/tags`/`GET /api/projects` reads and the `ReindexIn`-bodied `POST /api/reindex`.

### Accounts control plane (P10, new)

- **`persistence/`** — the ORM foundation. `base.py`: a `NAMING_CONVENTION` + declarative `Base` (stable Alembic constraint names). `models.py`: the **six** accounts tables (`UserModel`, `TenantModel`, `TenantMemberModel`, `ProjectModel`, `ProjectCredentialModel`, `AuthTokenModel`) — UUID PKs, tz-aware `created_at`; ownership is the `tenant_members` join (no owner column on `tenants`). `engine.py`: `get_engine`/`get_session_maker`/`dispose_engine` — lazy async singletons over `config.database_url()`; unset → `get_engine()` raises `RuntimeError` (dormant). Alembic lives at repo root (`alembic.ini` + async `alembic/env.py`, initial migration `0001_accounts_tenancy`); migrations run **explicitly** (`alembic upgrade head`), never on boot.
- **`accounts/`** — the layered accounts service (ported from the `vocky` reference): `security.py` (argon2id `hash_password`/`verify_password`, `generate_opaque_token` = `secrets.token_urlsafe(32)`, `sha256_hex`); `types.py` (transport-neutral `@dataclass` records — credential/session records **omit `token_hash`** so hashes never cross the transport boundary); `repository.py` (the **sole** ORM boundary — `add → flush → refresh → *Record`, never commits); `service.py` (owns transactions + domain errors `AccountsPersistenceError`/`AccountsReadError`/`DuplicateEmailError`; `create_tenant_with_owner` is the atomic signup primitive). `auth.py`: the reusable `require_user(request) -> AuthContext(user, tenant)` FastAPI guard (bearer → `sha256_hex` → active session token → user → `tenants[0]`) + `extract_bearer_token` + the generic-401 `auth_error_handler`. Entrypoint: `get_accounts_service()`.
- **`auth_api.py`** / **`app_api.py`** — two `APIRouter`s mounted by one `include_router` line each in `main.py`. `auth_api`: `/auth/signup|login|logout|me` (30-day opaque session tokens, enumeration-safe generic 401s, hash-free serializers). `app_api`: `require_user`-guarded `/app/*` (tenant, projects, per-project `vk_` credentials), all scoped to `ctx.tenant.id`; `_load_scoped_project` returns 404 for both missing and cross-tenant so existence never leaks. `vk_` mint recipe: `key=f"vk_{generate_opaque_token()}"`, `token_prefix=key[:12]`, `token_hash=sha256_hex(key)`, raw key returned once.
- **`api_auth.py`** — the `/api/*` two-mode credential resolver. `_tenant_mode()` switches per-call on `DATABASE_URL`. Legacy mode reproduces the old `require_bearer`/`require_read_bearer` guards byte-for-byte (returns the shared `_LEGACY` context, `tenant_id=None`). Tenant mode maps a bearer → `ApiAuthContext(tenant_id, project_id, is_public)`: exact `KB_API_TOKEN` → tenant #1 (the pinned master, via `get_tenant_one_id()`), a `vk_` key → its project's tenant (+project), a session token → the user's `tenants[0]`; unresolvable/absent → generic 401. `get_tenant_one_id()` resolves tenant #1 from `KB_OPERATOR_EMAIL` (normalized `.strip().lower()` — **P10.F1**, casing-tolerant) and caches **on success only** (so the operator seeded after boot still resolves). `is_public` marks legacy/tenant-#1 callers (→ `docs/`) apart from every other tenant (→ `tenants/<uuid>/`).
- **Per-tenant content routing in `main.py`** — `_tenant_root(ctx)` (public → `config.docs_root()`, else `KB_ROOT/tenants/<uuid>/`), `_tenant_db_id(ctx)` (write stamp: `''` legacy, else the uuid), `_tenant_filter(ctx)` (read scope: `None` legacy, else the uuid string). Git publish + project landing + Recent index run **only** for the public root (`ctx.is_public`); every read/search/list/by-id/by-path/delete query passes `tenant_id=_tenant_filter(ctx)`. `POST /api/reindex` is now `async`, resolves tenant #1 for the `docs/` walk, and stays operator-only (a `vk_` key → 401). `reindex.py` re-derives `tenant_id` from the content root so a rebuilt DB re-populates it from the path alone (hard coupling), and its vanished-row cleanup is tenant-scoped (one tenant's reindex never deletes another's rows).
- **`seed.py`** (`python -m server.seed`) — idempotent, Postgres-only operator/tenant/project seed built on `AccountsService`: creates the operator user (argon2id, race-safe on `DuplicateEmailError`) + tenant #1 (`create_tenant_with_owner`) + a `projects` row per **live `docs/` project derived from the tree** (reindex's `_FILENAME_RE` + `RESERVED_DIRS`, `project = rel.parts[0]`) — never hardcoded, so seeded names line up with `documents.project`. No `vk_` seeded (the master bearer covers tenant #1). `config.operator_email()`/`operator_password()` are the new seed-only settings.

### Usage-metering plane (P11, new)

- **`usage/` package (transport-neutral, mirrors the accounts layering).** `types.py`: the shared `EVENT_*` constants (`document.created`/`document.deleted`/`search` — imported by the hook and the read API, never re-declared) + the `@dataclass` records (`RecordUsageEvent` input; `UsageMetrics`/`UsageTotals`/`UsageDailyCount` aggregate). `repository.py`: the sole ORM boundary — `insert_usage_event` (add+flush, never commits) and `get_usage_metrics`, the derive-on-read aggregate (one grouped SELECT over `func.date(func.timezone("UTC", occurred_at))`, per-`event_type` `func.count().filter(...)`, window filtered half-open `occurred_at >= start AND occurred_at < end`; days zero-filled by `_iter_days` — which excludes a trailing exactly-midnight `end` day — and totals summed in Python). `service.py`: `UsageService` owns the session/transaction and domain errors — `record_event` runs on its **own isolated transaction and raises** `UsagePersistenceError` on failure (so best-effort lives at the call site, not buried), and `get_usage_metrics` surfaces `UsageReadError`. Entrypoint `get_usage_service()`.
- **`usage/metering.py` — the content-plane → usage-events bridge.** `UsageHint{tenant_id, event_type, project_name, project_id, credential_id}` is what a metered handler stashes on `request.state.usage`; `async record_usage(hint)` is **best-effort** (a broad `try/except Exception` + `logging.warning(exc_info=True)` around S1's raising `record_event`), so a metering failure never surfaces into the observed request. It resolves project attribution (`project_name` → tenant project UUID via `get_project_by_name`, else the `vk_` caller's bound `project_id`, else tenant-level NULL), records the event, then stamps `last_used_at` — **on the metered event only**, never in the resolver (stamping in the resolver would write on every read, contradicting "open reads stay fast"). A `vk_` key used only for reads therefore reflects its last write/search.
- **The stash + async middleware seam (`usage_metering` in `server/main.py`).** The metered content handlers (`create_document`, `delete_document`/`_by_path`, `search`) are **sync `def`** (they hold `WRITE_LOCK` over a sqlite critical section) and cannot `await` a Postgres write, and the event type + success status are only known *after* the handler runs. So each stashes a `UsageHint` on its **success path only**; a single `@app.middleware("http")` reads it off `request.state` after `call_next` and calls `record_usage` **iff** a hint is present, `hint.tenant_id is not None`, **and** the response is **2xx**. Error paths (401/404/409/422/400) raise before the stash, so only successes are metered; legacy mode carries `tenant_id=None` so the guard skips and no engine is created (inert). Awaiting in the middleware keeps metering synchronous w.r.t. the client while the sync handler never blocks on Postgres.
- **Supporting additions.** `ApiAuthContext.credential_id: UUID | None` (`server/api_auth.py`), set **only** in the `vk_` branch of `_resolve_tenant_bearer` (master/session leave it `None`), carried so the middleware can stamp the credential. `AccountsService.get_project_by_name(tenant_id, name)` (repository + service) — tenant-scoped, **oldest-wins** (`ORDER BY created_at, id LIMIT 1`, since names aren't unique per tenant) — for name→UUID attribution. `touch_credential_last_used` (built in P10, unwired) is now called by `record_usage`.
- **`usage_api.py` (read router).** `GET /app/usage` + `GET /app/projects/{id}/usage` over `get_usage_metrics`, `require_user`-guarded and tenant-scoped, reusing `_load_scoped_project`/`serialize_project`/`serialize_credential` from `app_api`. It only reports — no metering, no persistence. See api.

### HTML explainer document format (P16, new)

P16 adds a standalone self-contained HTML explainer as a first-class content-plane format, additive across the write path, storage, extraction, indexing, and a raw read route.

- **The body rule (the core invariant — identical on write and reindex).** `body` = the on-disk content minus its frontmatter block. Then: **md** → DB `markdown` = body, `raw_html` = NULL, `format` = `'md'`; **html** → DB `markdown` = `extract_html_text(body)`, `raw_html` = body, `format` = `'html'`. Both `create_document` and `reindex._index_file` apply this same rule against the same normalized body string, so a **fresh-DB reindex reproduces the write-path row byte-for-byte** (`markdown` **and** `raw_html`) — proven at review (wipe `kb.sqlite3` + `reindex()` → identical rows). Because the DB `markdown` column always holds readable text, FTS5 / `snippet()` / embeddings (`document_input`) run **unchanged**.
- **`documents.py` additions.** `rel_path(project, date, slug, fmt="md")` → `.html` for html (default keeps every existing caller byte-identical); `validate_related` accepts `.md` **or** `.html`; `serialize_html_frontmatter`/`parse_html_frontmatter` — a **leading `<!--kb\n<yaml field lines>\n-->\n` HTML-comment frontmatter block** (the browser ignores it; reindex re-derives title/date/tags/related/source from it) parallel to the `---`-fenced markdown frontmatter (factored to a shared `_frontmatter_inner_lines`, verified byte-identical to the pre-P16 markdown output); `extract_html_text(html)` — a stdlib `html.parser.HTMLParser` subclass that drops `script`/`style`/`template`/`noscript` content, inserts coarse block-boundary newlines, and whitespace-normalizes (**no new deps**); `write_document_file(..., fmt="md")` writes `serialize_html_frontmatter + "\n" + _normalize_body(body)` for html and re-parses its own output (self-roundtrip preserved). The md write path is byte-identical.
- **`db.py`.** `_SCHEMA` `documents` gains `format TEXT NOT NULL DEFAULT 'md'` and nullable `raw_html TEXT`; `init_db` back-fills both via the existing PRAGMA-guarded `ALTER TABLE ADD COLUMN` pattern (the `related` precedent); `upsert_document` gains keyword-only `format="md"` / `raw_html=None`, threaded through the INSERT columns **and** the `ON CONFLICT … DO UPDATE SET`. The FTS definition is untouched.
- **`main.py`.** `DocumentIn.format: Literal["md","html"] = "md"` (bad value → free 422); `create_document` applies the body rule, threads `format`/`raw_html` into the upsert, and adds an additive `format` to the 201 response. The `_INTERNAL` drop-set gains `raw_html` (never in `/api` JSON; `format` **is** exposed). Embeddings input already keys on the stored extracted text — unchanged.
- **`reindex.py` / `seed.py` widening.** `_FILENAME_RE` → `\.(md|html)$`; `_walk_root` globs a merged sorted `*.md` + `*.html`; `reindex_path`'s extension check widens to `.md`/`.html`; `_index_file` branches on extension → the matching frontmatter parser + the same body rule → upsert with `format`/`raw_html`. Malformed comment-frontmatter takes the same skip-with-reason path as md. `seed._discover_projects` auto-widens via the shared `_FILENAME_RE`, so an HTML-only project is discovered. The delete path is `rel_path`-keyed → already format-agnostic.
- **`documents_api.py` — the raw route.** `GET /app/documents/{doc_id}/raw` (`require_user`, tenant-scoped, `_DROP` gains `raw_html`): **404** when missing / cross-tenant / `format != "html"` / empty `raw_html`; else `Response(raw_html, media_type="text/html; charset=utf-8")` with `Content-Security-Policy: sandbox allow-scripts; frame-ancestors 'self'` + `X-Frame-Options: SAMEORIGIN` + `X-Content-Type-Options: nosniff` + `Cache-Control: no-store`. The served bytes exclude the comment-frontmatter (start at `<!DOCTYPE html>`). Unmetered. The XSS-containment rationale is in **architecture**; the web relay in **frontend**.
- **Accepted quirk (recorded, not engineered around).** Overwriting a slug with the other format yields a **different `rel_path`** (the extension differs), so an `.md` and an `.html` file can coexist for the same project/date-slug; delete is per-`rel_path`, so this is consistent.

### Accounts v2: org-level credentials + get-or-create (P18, new)

P18 makes org-level `vk_` keys honest and projects get-or-create, all on the P18.S1 schema (`0003`; see data). Additive across the accounts service, the resolver, and the write path.

- **`provision_signup(user_id)` — the shared signup primitive** (`server/accounts/service.py`). Creates tenant `"default"` + an `owner` `tenant_members` row + project `"default"` in **one transaction**, returning `(TenantRecord, TenantMemberRecord, ProjectRecord)`. Constants `DEFAULT_ORG_NAME` / `DEFAULT_PROJECT_NAME` (both `"default"`) live beside it. Signup (`server/auth_api.py`) and the **fresh-DB** seed (`server/seed.py`) both call it, so signup and a first-boot seed can't drift; `create_tenant_with_owner` is retained for bare-tenant callers. **Seed no longer renames an existing tenant** — an already-seeded prod tenant #1 keeps its name and only gains a `"default"` project lazily (via get-or-create). Signup's response gained an additive `project` field alongside `tenant`.
- **Resolver reads `cred.tenant_id` directly** (`server/api_auth.py` `_resolve_tenant_bearer`). An **org-level** key (`project_id NULL`) resolves to `ApiAuthContext(tenant_id=cred.tenant_id, …)` with **no project query**; a **project-bound** key keeps the vanished-project existence guard (`get_project(cred.project_id)`) and carries `project_id` for attribution. Master-token + session paths are untouched. This is **no trust-boundary change** — enforcement was already tenant-wide; org-level keys make it honest (see security).
- **New service/repository methods.** `get_or_create_project(tenant_id, name)` — `get_project_by_name` first, else `create_project` + commit, **race-safe** against the new `UNIQUE(tenant_id, name)` (catch `IntegrityError` → rollback → re-read). `list_org_credentials(tenant_id)` — `WHERE tenant_id = … AND project_id IS NULL`, oldest-first. `serialize_credential` is now **NULL-safe** (`project_id: null` for org keys) — it is the shared serializer `usage_api` also imports.
- **Write-path get-or-create via an async dependency.** `ensure_registry_project(body, ctx)` (async) runs `get_or_create_project(ctx.tenant_id, body.project)` and returns the id; it is wired as a FastAPI `Depends` in front of the sync, `WRITE_LOCK`-holding `create_document` (FastAPI parses the body once and caches `resolve_api_write`, so the dep runs once, before the handler — the seam that lets a Postgres write precede the sqlite critical section). An unregistered `body.project` now always leaves a `projects` row (idempotent, harmless even if the handler later 409/422s). `POST /app/projects` was folded into `get_or_create_project` too (a duplicate name → 201 existing row, closing the latent dupe-500 the new UNIQUE would otherwise cause). `server/usage/metering.py` is **unchanged** — its name-first `get_project_by_name` lookup now always finds the row the write get-or-created.

## Invariants

- **Single-writer**: the write lock is **in-process**, so the API must run **one uvicorn worker** — never scale workers. **(P10)** the async Postgres accounts plane sits alongside but never touches `WRITE_LOCK`; the single-worker invariant is unchanged.
- **(P10) Dormant control plane / legacy parity**: with `DATABASE_URL` unset the accounts engine is never created and `/api/*` is byte-for-byte the pre-P10 single-`KB_API_TOKEN` deployment (the 65-test legacy regression gates this).
- **(P11) Metering is best-effort and never fails a request**: the sync handlers only *stash* a hint; the async middleware records post-response, on a 2xx with a hint and `tenant_id` set, and `record_usage` swallows every error (logged, never raised). In legacy mode the hint carries `tenant_id=None`, the guard skips, and no engine is created — so metering is fully inert and the frozen `/api/*` contract is unchanged (same 65-test legacy regression). The open read/list path stays **unmetered** (no `request.state.usage` stash) so it holds no hot-path write.
- **(P10) Tenant identity lives in the file path**: `reindex` re-derives `tenant_id` from the content root (`docs/` → tenant #1; `tenants/<uuid>/` → the dir name), so cross-tenant isolation survives a full disposable-DB rebuild. WAL still gives read concurrency. P4's embedding work introduces no background workers — embeds happen in-request (best-effort, outside the lock) or at reindex, so the single-worker invariant is untouched.
- **`docs/`-canonical, never-rollback**: a failed git commit leaves the file/DB written (`committed:false`); `docs/` stays canonical and reindex reconciles. Both the POST write path and the P4 DELETE path share this contract. Git stages only the touched paths and never pushes.
- **DB/file consistency**: a POST-written document and a later reindex produce identical stored `markdown` — no phantom drift. `related` and the sanitized `source_repo` round-trip identically through the write path and reindex.
- **Embeddings are a disposable cache, best-effort everywhere**: no Gemini key → skipped; an API/429 failure → per-doc skip, reported and retried next reindex — never fails a reindex, a write, or a search (which degrades to BM25-only).

## Error Handling

- `ConventionError`/`FrontmatterError` (incl. `validate_related` shape) → 422; an invalid `reindex_path` rel_path → 422; existing target → 409 (names the doc); a missing delete target → 404; FTS5 raw-syntax error → 400 (`SearchQueryError`); git failure → 201/200 with `committed:false` + `commit_error`.
